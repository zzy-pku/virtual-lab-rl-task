# 教程 3：第一次标准训练

本教程的目标是：

> **在完成 Level 1 或 Level 2 的准备后，真正独立完成一次标准训练，并保存最基本的结果。**

这份教程默认你已经：

- 拥有可运行环境（Docker 或 uv 本地环境）；
- 准备好了 π0.5 模型；
- 具备标准 benchmark 的运行条件；
- 能执行项目训练命令。

---

## 1. 训练前先明确这四件事

在按下回车前，请先把下面四件事写在你的实验记录里：

1. 我这次训练用的是哪个 benchmark / 任务；
2. 我用的是哪个模型 checkpoint；
3. 我用的是哪个配置文件；
4. 我这次最想观察什么（例如：训练是否稳定、success rate 是否上升）。

如果这四件事你自己都说不清楚，那么这次训练大概率会变成“黑箱跑数”。

---

## 2. 先不要修改太多配置

第一次标准训练的原则是：

> **先跑通，再改。**

所以第一次运行时：

- 不要自己大改算法；
- 不要同时修改多个关键超参数；
- 不要同时更换 benchmark、任务和模型；
- 先尽量靠近教学给定的标准设置。

这样你后面做对照实验时，才知道基线是什么。

---

## 3. 训练时需要盯住哪些东西

训练启动后，至少盯住下面几类信息：

### 1）环境是否真的在交互
看日志里是否有 rollout、环境步数、episode 更新等迹象。

### 2）训练是否真的在更新
看是否有 loss / reward / success / eval 相关量持续刷新。

### 3）有没有明显异常
例如：
- 一启动就 loss 爆掉；
- reward 长时间完全不动；
- success rate 始终是 0；
- 训练中断或显存问题。

### 4）日志输出目录
确认训练产生的文件放在哪，后面分析要用。

---

## 4. 第一次训练后至少要保留哪些结果

第一次训练结束后，请至少保存：

- 训练命令
- 使用的 config
- checkpoint 路径
- 日志目录
- 一张关键指标曲线图
- 一段 5~10 行的结果说明

这个“结果说明”至少应回答：

1. 训练有没有正常运行；
2. 最核心的结果指标是什么；
3. 当前结果看起来是有效、无效，还是还无法判断；
4. 如果继续做下一次实验，你最想改什么。

---

## 5. 做最小结果分析

你第一次训练不要求写长报告，但至少要做一个最小分析：

### 问题 1：reward 或 success 有变化吗？
如果有，变化方向是否合理？

### 问题 2：训练是否稳定？
有没有明显波动、崩掉、长期无改进等现象？

### 问题 3：这个结果说明了什么？
它说明项目真的通了，还是说明还有配置/环境问题？

这一步是你从“跑完训练”走向“理解训练”的第一步。

---

## 6. 第一次训练后推荐做的一个最小对照实验

第一次标准训练完成后，请马上做一个最小对照实验。

最推荐的做法是只改一个变量，例如：

- 训练步数；
- PPO 的一个关键超参数；
- 任务子集；
- observation 组合；
- 相机视角设置。

注意：

- 一次只改一个变量；
- 记录修改原因；
- 比较结果变化；
- 写出你的解释。

如果你一下改太多参数，后面就不知道是哪一个导致了变化。

---

## 7. 完成本教程的标志

当你完成下面四件事时，就算真正完成了第一次标准训练：

- 跑通一组标准实验；
- 保存训练日志和关键结果；
- 做了一次最小对照实验；
- 写出了最短结果说明和现象解释。

---

## 8. 下一步怎么走

- 如果你还不熟练：继续在 Level 1 / Level 2 做一次新的对照实验；
- 如果你已经能稳定做标准训练并解释结果：准备进入 Level 3；
- 进入 Level 3 前，先自己尝试回答：
  - 我能不能换一个 benchmark 子任务自己组织配置？
  - 我能不能自己写一个实验计划，而不是照抄标准例子？

这两个问题都能回答得比较清楚时，说明你已经接近 Level 3。
