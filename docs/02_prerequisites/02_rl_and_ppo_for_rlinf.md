# 前置知识：RL 与 PPO（面向 RLinf 任务的最低必要版本）

本文件不是完整强化学习课程，而是面向 RLinf 任务线的“够用版”补课材料。

目标：让你在做 RLinf + π0.5 任务时，至少知道训练在优化什么、日志里在看什么、参数变化大致会影响什么。

---

## 1. 什么是强化学习（在本任务里）

在当前 RLinf 任务中，你可以把强化学习理解成：

- 模型根据当前观测和任务描述输出动作；
- 环境执行动作后返回结果；
- 结果通过 reward 反馈给训练算法；
- 算法根据 reward 调整策略，使未来动作更有可能完成任务。

在这里你不需要先背完整数学定义，只要先抓住：

> **RL 是“边试边学”的策略优化过程。**

和监督学习不同，这里通常没有直接告诉模型“标准动作标签是什么”，而是通过环境反馈来告诉模型“这样做是更好还是更差”。

---

## 2. 这条任务线里的几个核心对象

### policy（策略）
当前可以简单理解为：

> 给定观测和任务描述，模型决定接下来输出什么动作。

在 RLinf + π0.5 任务中，π0.5 就是承担策略角色的核心模型。

### observation（观测）
环境给模型的信息。当前任务里通常包括：

- RGB 图像
- 可能还包括状态信息
- 自然语言任务描述

### action（动作）
模型输出给环境的控制量。当前典型任务中是连续动作，例如：

- 末端位置控制
- 旋转控制
- gripper 开合

### reward（奖励）
环境对动作结果的反馈。你可以把它理解成“当前这一步做得好不好”的信号。

### rollout
指模型在环境里连续执行若干步并收集经验的过程。训练不是只看单一步动作，而是要看一整段交互轨迹。

---

## 3. PPO 在这里大概做什么

你现在不需要完整掌握 PPO 的推导，但必须知道它在任务里扮演什么角色。

PPO 的目标可以粗略理解成：

> 在不让策略变化过于激进的前提下，逐步把策略推向更高 reward 的方向。

为什么需要“不要变化太激进”？
因为如果每次都更新太猛，训练会非常不稳定，可能突然崩掉。

所以你可以先记住 PPO 的三个直觉：

1. 它会利用 rollout 收集来的经验更新策略；
2. 它会尽量让“新策略比旧策略更好”；
3. 它会限制每次更新不要过大。

这就足够支撑你阅读 Level 1 / Level 2 的训练日志和配置。

---

## 4. 你在日志里通常会看到什么

在开始训练之后，你常会遇到下面几类量：

### reward 或 episode return
反映当前策略在任务中的总体表现。它升高通常说明策略在变好，但不能只看这一项。

### success rate
很多机器人任务比 reward 更直观的指标。它表示任务真正成功的比例。

### loss
不同组件的优化目标值。不要简单理解为“越低越好”，而要结合任务表现一起看。

### kl / clip / entropy 等 PPO 相关量
这些量更多反映训练是否稳定、更新是否过猛、策略是否过早塌缩。
你现在的最低要求是：

- 知道它们是“训练状态指示器”；
- 知道如果曲线异常波动，可能说明训练不稳定。

---

## 5. 在 Level 1 / Level 2 中你最需要理解的不是公式，而是这三件事

### 第一：训练输入是什么
图像、语言任务描述、状态信息等是怎么进入模型的。

### 第二：训练在优化什么
模型动作通过环境得到 reward，然后 RL 算法更新策略。

### 第三：结果怎么看
训练结束后，不能只说“跑完了”，而要能判断：

- reward 有没有明显变化；
- success rate 是否提升；
- 不同配置下结果为什么不同。

在进入 RLinf 任务前，请确认你能回答下面问题：

- PPO 的作用大概是什么？
- 训练日志里 reward / success rate 大概怎么看？
